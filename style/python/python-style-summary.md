# python-style-summary

## 開発環境

- パッケージマネージャー: uv
- Pythonバージョン: 3.13

## アーキテクチャ・パッケージ化

- パッケージ化対象: ファイル数が4つ以上、かつ内部インポート（密結合）が発生する場合、またはポートフォリオ配布用（srcレイアウトを採用）。
- スクリプト維持: ファイル数が3つ以下、または独立したサブプロセスとして外部から叩くだけの疎結合な構造の場合。

## コードスタイル・命名規則

- パッケージ / モジュール / 関数 / 変数: snake_case
- クラス: PascalCase
- 定数: UPPER_SNAKE_CASE
- 内部用（プライベート）: 先頭にアンダースコア（\_snake_case）
- インデント: スペース4つ
- 空白行: クラス間は2行、関数間は1行、ロジックの区切りは1行
- 一行の長さ: 制限なし

## ドキュメンテーション・型ヒント

- Docstring: Googleスタイル（ファイル先頭、クラス、関数に必須）
- 型ヒント: Python 3.13標準の組み込みジェネリクスを使用（例: list[str], dict[str, int], int | None）

## インポート順序

1. 標準ライブラリ
2. サードパーティ製ライブラリ
3. 自作モジュール / ローカルライブラリ

- 各グループ内はアルファベット順に整列すること。

## ロギング・エラー処理

- ログ出力: `print()` は禁止。標準の `logging` モジュール（`getLogger(__name__)`）を使用。
- エラー処理: 汎用的な `Exception` のみによるキャッチは避け、具体的な例外を指定。リソース操作は必ず `with` 構文を使用。

## 設定管理・環境変数

- CLI引数: 実行ごとに変更する動的パラメータ。
- config.py: 固定のデフォルト値、パス、数値パラメータ、プロンプトの管理。環境変数の読み込みを内包する。
- .env: APIキーなどの機密情報、ローカル環境依存のパス。
- 実装方法: Pydantic-settings（SettingsConfigDict）を推奨。または pathlib と python-dotenv を組み合わせ、ファイルの存在チェックと絶対パス解決を必須とする。

## CLIフレームワーク

- 小〜中規模: argparse
- 大規模: Typer（typing.Annotated 構文を使用）
