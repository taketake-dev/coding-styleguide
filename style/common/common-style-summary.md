# common-style-summary

## コーディングスタイル

- UTF8
- 改行: LF
- 最後は空行

## コメント

- AI特有のフレンドリーな表現や絵文字は不要

## Print、画面への表示

- logを使う
- 日本語
- 最小限

### 文章のルール

- 日本語（です・ます調）
- 形式は [機能/対象] 動作の結果 (補足情報)
  - 例：[ユーザー認証/ログイン] 認証に成功しました (セッションを開始します)

### loggingのルール

出力する情報の重要度に応じて以下の5つのログレベルを使い分けること。

- DEBUG
- INFO
- WARNING
- ERROR
- CRITICAL / FATAL

以下の4つの要素を同一の並び順で含める

- タイムスタンプ
  - 形式：ISO 8601形式
- ログレベル
  - 形式：大文字表記
- ソースコンテキスト
  - 形式：{モジュール名 | クラス名 | ファイル名と行数}
- メッセージ
  - 形式：発生した事実を簡潔に
